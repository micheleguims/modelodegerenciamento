# 📊 Admin Dashboard — Gestão de Registros e Indicadores

Aplicação web desenvolvida para gerenciamento de documentos e acompanhamento de indicadores operacionais em uma interface moderna, responsiva e orientada a usabilidade.

Este projeto tem como foco demonstrar a construção de um **painel administrativo completo**, com controle por perfil de usuário, persistência de dados em tempo real e organização de fluxos de informação.

---

## 🎯 Objetivo do Projeto

Construir uma aplicação demonstrativa capaz de simular cenários reais de operação administrativa, explorando:

- **Gerenciamento estruturado de dados**: organização de registros com status, anexos e categorização;
- **Interface responsiva e intuitiva**: foco na experiência do usuário;
- **Controle de acesso por perfil**: diferenciação entre usuário administrador e operador;
- **Atualização em tempo real**: integração com banco de dados dinâmico;
- **Arquitetura escalável**: separação de componentes e reutilização de lógica.

---

## 🛠️ Stack Tecnológica

O projeto utiliza tecnologias modernas voltadas para desenvolvimento de aplicações web:

- **Front-end**: React + Tailwind CSS
- **Backend / Banco de Dados**: Firebase Firestore (tempo real)
- **Visualização de dados**: Recharts
- **Ícones**: Lucide React

---

## ✨ Funcionalidades Principais

- 📊 **Dashboard Operacional**  
  Visualização de métricas e indicadores de registros em tempo real.

- 📁 **Gerenciamento de Registros (CRUD)**  
  Criação, edição, exclusão e organização de documentos com suporte a anexos (PDF e planilhas).

- 🔍 **Filtros por Unidade/Setor**  
  Segmentação dos dados para análise mais precisa.

- 🔐 **Controle de Perfis**
  Acesso diferenciado entre administrador e operador.

- 🧾 **Histórico de Atividades**  
  Registro de ações realizadas pelos usuários para rastreabilidade.

## 💻 Instalação e Uso Local

1. Clone o repositório:
```bash
git clone https://github.com/micheleguims/NOME-DO-REPO.git
```

2. Instale as dependências:
```bash
npm install
```

3. Configure o Firebase:
Crie um arquivo .env na raiz com suas chaves de API (verifique o .gitignore).

4. Inicie o projeto:
```bash
npm start
```
